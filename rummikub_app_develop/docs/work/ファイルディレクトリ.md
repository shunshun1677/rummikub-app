## トップレベルディレクトリの責務

| ディレクトリ | 責務 | 配置例 |
| --- | --- | --- |
| `assets` | アセットファイルを配置する | アイコン、画像、フォントファイル |
| `components` | ページを跨いで利用される共通コンポーネントを配置する | ボタン、ラベル、共通レイアウト |
| `features` | 各モジュールや機能ごとのファイルを配置する | アカウント機能、ユーザー機能 |
| `infrastructures` | 外部リソースへの実アクセス実装を配置する | API 定義、Firebase、Elasticsearch |
| `pages` | ルーティングされた画面ファイルを配置する | アカウント一覧ページ、アカウント詳細ページ |
| `stores` | アプリケーション全体でグローバル管理される情報を配置する | ログインアカウント情報 |
| `utils` | 言語特有の汎用ユーティリティを配置する | 配列処理、日付処理 |

## `components` の責務

`components` には、ページを跨いで利用される共通コンポーネントを配置します。

特定のページ内でしか利用しないコンポーネントは、`features` 配下の対象機能ディレクトリに配置します。

### 設計ルール

- コンポーネントは `Container` 層と `Presentation` 層に分ける
- `Presentation` 層は View のレンダリングに関わるソースのみを管理する
- `Container` 層は `hooks` や `stores` にアクセスし、取得した値や処理を `Presentation` ファイルへ渡す

### サブディレクトリ

| ディレクトリ | 責務 | 配置例 |
| --- | --- | --- |
| `components/mui` | Material UI コンポーネントを wrap したシンプルなコンポーネントを配置する | テキスト、ボタン |
| `components/composite` | Material UI コンポーネントを組み合わせた共通コンポーネントを配置する | 確認ダイアログ、ユーザープロフィールカード |
| `components/layout` | page を構成するためのデザインフレームを提供する | 詳細ページ用レイアウト、一覧ページ用レイアウト |

## `features` の責務

`features` には、各モジュールや機能ごとのファイルを配置します。

機能ごとに `features/feature_a/` や `features/feature_b/` のようにディレクトリ階層を分けます。さらに機能内のサブモジュールは、`features/feature_a/sub_feature_c/` のように管理します。

モジュール内で共通利用されるファイルは、`features/feature_a/_shared/` に配置します。

### サブディレクトリ

| ディレクトリ | 責務 |
| --- | --- |
| `components` | feature 内で利用するコンポーネントを配置する |
| `dict` | feature 内で利用する辞書ファイルを配置する |
| `hooks` | feature 内で利用するカスタムフックを配置する |
| `logics` | feature 内で利用する共通ロジックを配置する。React のカスタムフックに依存しない処理を置く |
| `models` | feature 内で利用するモデルクラスを配置する。モデル特有のドメインロジックを内包する |
| `repositories` | feature 内で利用するリソースへのアクセスクラスを配置する。内部では `infrastructures` の実装ファイルを呼び出す |
| `stores` | feature 内で利用する状態を持つファイルを管理する。本プロダクトでは主に context を利用する |
| `types` | feature 内で利用する型定義ファイルを管理する |

## `pages` の責務

`pages` には、ルーティングされた画面ファイルを配置します。

ディレクトリは URL のパスと一致させ、`features` 配下の対応するコンポーネントファイルとの繋ぎ込みを行います。

## `stores` の責務

`stores` には、アプリケーション全体でグローバル管理される情報を配置します。

特定の機能やページ内でグローバル管理したい情報は、`features` 配下の `stores` で管理します。

## `utils` の責務

`utils` には、言語特有の汎用ユーティリティを配置します。

特定の機能に関する共通処理は、`features` 配下の `logics` で管理します。

# 追加要望

実際にユーザーにラミーキューブアプリを使用してもらった結果として出てきた要望、および今後追加で出てきそうな潜在要望を整理する。

---

## 1. 実際に出てきた要望

### 1-1. ゲーム終了時の情報公開

* ゲーム終了時に、CPUの手札を公開したい。
* 勝敗が決まったあと、CPUがどのような牌を残していたのか確認できるようにしたい。

### 1-2. ドローした牌の視認性向上

* 自分が山札から引いた牌が何だったのか分かるようにしたい。
* 例として、直近で引いた牌に赤枠を付けるなど、他の牌と区別できる表示にしたい。

### 1-3. 場のカード並び替え

* 場に出ているカードを自動でソートしたい。
* または、セット内でドラッグアンドドロップにより順番を入れ替えられるようにしたい。
* run の並び順が崩れている場合でも、ユーザーが自然に整理できるようにしたい。

### 1-4. 「1枚引いて終了」の誤操作防止

* 「1枚引いて終了」を間違えて押下したときに、場に出したカードが無かったことにされてしまう問題を防ぎたい。
* 自分が場にカードを1枚でも出した場合は、「1枚引いて終了」を非活性にするなどの制御を入れたい。
* 少なくとも、場に出した操作がある状態では誤ってターン終了しにくいUIにしたい。

### 1-5. カード文字色の改善

* カードの文字色が黄色だと見にくい。
* 黄色ではなく、緑色など視認性の高い色に変更したい。

### 1-6. 操作方法のドラッグアンドドロップ化

* カードの選択や場に出す操作は、ドラッグアンドドロップのみで行えるようにしたい。
* セットごとに設けられている「選択タイルを追加」は不要にしたい。
* セットごとに設けられている「空セット削除」は不要にしたい。

### 1-7. 不要な表示の削除

* 「選択中」の表示は不要にしたい。
* 「新規セット」の表示は不要にしたい。
* 操作中の状態を表示しすぎず、より直感的に扱える画面にしたい。

### 1-8. 新規セット種別の自動判定

* 「新規セット」が run なのか group なのかを、ユーザーが明示的に選ばなくてもよいようにしたい。
* 場に出したときに、run または group として成立しているかを自動で検知したい。
* run または group として有効な場合は、そのまま場に残るようにしたい。
* 無効な組み合わせの場合は、場に置けない、または未確定状態として分かるようにしたい。

### 1-9. 手牌ソート方法の追加

* 自分の手牌を「カラー + 数字の昇順」だけでなく、「数字のみの昇順」でも並び替えられるようにしたい。
* 数字を軸にして、色違いの同じ数字を見つけやすくしたい。

### 1-10. 場が増えたときの操作性改善

* 場のセットが増えると画面が縦に伸び、操作しづらくなる。
* 画面全体のズームを縮小する必要が出てしまうため、アプリ側で見やすく操作しやすいレイアウトにしたい。
* 盤面が広がっても、手牌・場・操作ボタンを無理なく扱えるようにしたい。

---

## 2. 潜在的に出てきそうな要望

### 2-1. 操作の取り消し・復元

* 直前の操作を1手ずつ戻せるようにしたい。
* ターン開始時点へ戻すだけでなく、ドラッグ移動や並び替え単位で取り消したい。
* 誤操作が起きたときに、安心して盤面を編集できるようにしたい。

### 2-2. ターン終了前の確認

* 場に変更がある状態でターン終了やドロー終了を押した場合、確認ダイアログを出したい。
* 未確定の変更、無効なセット、初回30点未達などがある場合は、理由を表示してほしい。

### 2-3. 無効セットの分かりやすい表示

* run / group として成立していないセットを視覚的に分かるようにしたい。
* 無効な理由を簡単に表示したい。
* 例: 「3枚以上必要」「同じ数字で色違いにしてください」「連番になっていません」など。

### 2-4. 出せる候補のヒント表示

* 自分の手牌から出せる run / group の候補を提示してほしい。
* 初回30点を満たせるかどうかを補助表示してほしい。
* 初心者でも次の一手を考えやすいようにしたい。

### 2-5. CPUの行動ログ

* CPUが何をしたのかを確認できる履歴がほしい。
* 例: 「CPUが1枚引きました」「CPUが3枚出しました」「CPUがセットを組み替えました」など。
* CPUの手が分かりにくい場合に、ゲーム進行を追いやすくしたい。

### 2-6. ゲーム履歴・リプレイ

* 各ターンで誰が何をしたかを後から確認したい。
* ゲーム終了後に、勝敗までの流れを振り返れるようにしたい。

### 2-7. 表示サイズ・密度の切り替え

* 牌の表示サイズを大・中・小で切り替えたい。
* 場が広がったときはコンパクト表示に切り替えたい。
* 画面サイズに応じて、自動的に表示密度を調整してほしい。

### 2-8. モバイル・タブレット対応

* スマートフォンやタブレットでも操作しやすくしたい。
* タッチ操作でドラッグアンドドロップがしやすいUIにしたい。
* 小さい画面でも手牌と場を行き来しやすくしたい。

### 2-9. ルール説明・補助ガイド

* 初回30点、run、group、ジョーカーの扱いなどをアプリ内で確認できるようにしたい。
* 操作に迷ったときに、簡単なヘルプを開けるようにしたい。

### 2-10. 設定画面

* 文字色、牌サイズ、ソート方法、確認ダイアログの有無などを設定できるようにしたい。
* ユーザーごとの好みに合わせて操作感を調整したい。

### 2-11. 勝敗結果の詳細表示

* ゲーム終了時に、残り手牌の合計点を表示したい。
* プレイヤーとCPUの残り手牌、点数差、勝敗理由を分かりやすく表示したい。

### 2-12. アクセシビリティ改善

* 色だけに依存せず、形・ラベル・枠線などでも牌の状態を判別できるようにしたい。
* コントラストを高め、見やすい配色にしたい。
* キーボード操作にも対応できると望ましい。

---

## 3. 優先度の目安

### 高

* 「1枚引いて終了」の誤操作防止
* ドローした牌の視認性向上
* 場が増えたときの操作性改善
* 新規セット種別の自動判定
* ドラッグアンドドロップ中心の操作改善

### 中

* CPU手札のゲーム終了時公開
* 手牌ソート方法の追加
* 場のカード自動ソート・セット内並び替え
* 無効セットの分かりやすい表示
* カード文字色の改善

### 低

* CPUの行動ログ
* ゲーム履歴・リプレイ
* 設定画面
* ルール説明・補助ガイド

---

## 4. 実装時の注意点

* ドラッグアンドドロップ化する場合でも、ゲーム状態が破綻しないように、手牌・場・山札の状態管理を明確に分ける。
* 場に出した変更と、ターン終了時に確定された状態を分ける。
* 「1枚引いて終了」と「場に出してターン終了」は、ユーザーの行動履歴に応じて押せる条件を明確にする。
* run / group の自動判定は、ジョーカーを含むケースでも誤判定しないようにする。
* 視認性改善では、色覚差や画面輝度の違いを考慮し、色だけに頼らない表現も検討する。
* 盤面が広がる問題は、スクロール、折り返し、ズーム、固定手牌エリアなどを組み合わせて検討する。

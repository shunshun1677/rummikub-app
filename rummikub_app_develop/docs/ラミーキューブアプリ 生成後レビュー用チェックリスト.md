# ラミーキューブアプリ 生成後レビュー用チェックリスト

AIが出力した実装コードをレビューするためのチェックリスト。
目的は、**見た目ではなく、要件どおりに正しく遊べるか** を確認すること。

---

## 1. まず最初に見ること

### 1-1. プロジェクトとして成立しているか

* [ ] ファイル構成が提示されている
* [ ] 各ファイルの import / export が整合している
* [ ] 擬似コードではなく実コードになっている
* [ ] 起動手順が書かれている
* [ ] React + TypeScript 前提の構成になっている
* [ ] `any` が多用されていない
* [ ] 型定義が最低限そろっている

### 1-2. コンパイルできそうか

* [ ] 未定義の関数を呼んでいない
* [ ] 未使用の型名や不整合な型名がない
* [ ] import漏れがなさそう
* [ ] JSXの閉じ忘れや構文エラーがなさそう
* [ ] propsの受け渡しが噛み合っている

---

## 5. セット判定レビュー

### 5-1. run判定

* [ ] 同じ色で判定している
* [ ] 連番を見ている
* [ ] 3枚以上を条件にしている
* [ ] ジョーカーを欠番補完として扱っている

### 5-2. group判定

* [ ] 同じ数字で判定している
* [ ] 色重複を許していない
* [ ] 3〜4枚の条件を見ている
* [ ] ジョーカーを不足色補完として扱っている

### 5-3. ジョーカー対応

* [ ] run / group の両方でジョーカーを扱っている
* [ ] ジョーカーあり判定が雑すぎない
* [ ] ジョーカーなしケースと分けて考えられている
* [ ] ジョーカー回収ルールが勝手に入っていない
